Data preparation - Genetic Map (.map)

The aim is to create a "target genotype" (ideotype) with all the favorable alleles at the QTL positions. So, before running OptiMAS it is necessary to define the parental alleles to assemble (see table below). The map file is supplied by the user to specify the information regarding markers, QTL and identification of favorable alleles, issued from a QTL mapping analysis (possibly considering several traits, environments, etc).

The QTL position is rarely located at a marker; consequently QTL alleles are unknown and must be inferred from flanking markers. Thus, it is very important to select a subset of markers as informative as possible (especially in multi-parental context) to follow the favorable parental alleles (based on haplotypes). The number of markers selected per QTL should be in the range of 2-6 in order to avoid intensive computation time (more progress in this area will be made in the next version 2.0).

In this file, each QTL region is characterized by the estimated position of the QTL, the identification of the parent carrying the favorable allele and the names and positions of the markers that will be used to compute the allele transmission.

In red are columns/lines to add which differ from the Flapjack format.

Locus: name of markers/QTL. The name of the QTL must start with "qtl" (e.g. qtl_umc11, qtl_1 ...) and therefore the other names are by default considered as markers. The only constraint on names is to avoid blank in character chain.

Chr: index (numerical value) of the chromosome where the marker/QTL is located.

QTL: affiliation of markers to QTL (e.g. 4 markers Marker1/Marker2/Marker3/Marker4 are used to follow allele transmission for QTL 1). Indexes for each QTL must be unique, continuous (no gap in list) starting at 1 and in ascending order along the genome.

Pos: relative chromosomal position of loci. For QTL, this is the estimated QTL position coming from the QTL detection results. Positions of the different loci must be obtained using the Haldane's mapping function (i.e. by assuming no interference).

All+: identification of the parental allele(s) considered as being favorable. For QTL 1, the favorable allele "a" refers to the parental line named "IL1" (see columns "P1"/"P2" of the genotype/pedigree file). For QTL 2, "a/b" refers to parental lines (b and c) which can be considered as favorable relatively to other parental lines.

Note:

  • Never change the header field names (Locus/Chr/QTL/Pos/All+).
  • It is recommended to code parental alleles ("All+" column) by only one character (e.g. a, A, b, 1 ...).
  • Check that decimals are 0.00 and not 0,00 for the loci/QTL positions column ("Pos").
  • Both the map and the genotype/pedigree files must be in plain-text, tab-delimited format. So, use tabulation and not spaces between fields (e.g. marker1{tab}1{tab}1{tab}42.2 or qtl1{tab}1{tab}1{tab}70.0{tab}a).
  • The markers present in this map file must be ordered and match those in the genotype/pedigree file (same number of markers, only QTL are added).
